# poe-simulator — 프로젝트 규칙

## 프로젝트 개요
Path of Exile 한국 서버(카카오게임즈) 플레이어를 위한 시뮬레이터 웹사이트.
모든 사용자 노출 텍스트는 **카카오 공식 한글 번역**을 사용한다.

---

## 1. 번역 용어 규칙 (최우선)

### 기준 파일
- `docs/poe-glossary.json` — 영문 → 공식 한글 단순 매핑 (259개)
- `docs/poe-glossary-full.json` — 분류/비고 포함 전체 데이터

**한글 게임 용어를 작성하거나 수정할 때는 반드시 위 파일을 먼저 조회한다.**
기억이나 추측으로 번역하지 않는다.

### 용어집에 없는 용어
1. `https://poedb.tw/kr/` 에서 검색해 공식 한글명을 확인한다.
2. 확인되면 사용하고, **용어집 파일에 항목을 추가한다** (비고에 `[poedb 확인]` 표기).
3. 끝내 확인 불가하면 임의 번역 금지. `추정 번역(English Original)` 형태로 병기하고
   `docs/TODO-terms.md` 에 기록한다.

### 고유명사
유니크 아이템 / 스킬 젬 / 보스 이름은 확신이 없으면 영문을 병기한다.
잘못된 한글명은 거래소 검색을 방해하므로 치명적이다.

### 표기 원칙
- UI 라벨: 공식 명칭 사용 (예: "신성한 오브", "엑잘티드 오브")
- 커뮤니티 약칭(디바인, 엑잘, 카오스, 6링, CI, MoM)은 **검색/별칭(alias)으로만** 허용,
  기본 표시 텍스트로는 쓰지 않는다.

---

## 2. i18n 구조 규칙

- 한글 문자열을 컴포넌트에 **하드코딩하지 않는다.** 반드시 로케일 파일의 키를 통해 참조한다.
- 새 키를 추가하면 **모든 로케일 파일에 동시에** 추가한다 (`en`, `ko`).
- 키 네이밍: `<도메인>.<컴포넌트>.<의미>` (예: `currency.chaosOrb.name`)
- 게임 데이터에서 유래한 이름(아이템/젬/모드)은 UI 문구용 로케일이 아니라
  **게임 데이터 레이어에서 en/ko 쌍으로** 관리한다.

### 번역 누락 검사
작업 후 다음을 확인한다:
- `ko` 로케일에 없는 키 (fallback으로 영어가 그대로 노출되는 항목)
- 값이 영문 그대로인 `ko` 항목
- 용어집과 불일치하는 한글 표기

## 3. 게임 정보 최신성

PoE는 3~4개월마다 리그가 바뀌고 밸런스가 크게 변한다. 학습 지식은 구버전일 수 있다.

**다음 항목은 반드시 웹 검색 후 반영한다:**
현재 리그 이름/메커니즘, 아이템·젬 수치, 밸런스 변경, 아틀라스 구조, 시세.

**소스 신뢰도 순서:**
1. `poedb.tw/kr/` — 공식 한글 명칭 및 게임 데이터 기준
2. 카카오 공식 패치노트
3. `poewiki.net` (영문 — 용어는 poedb kr로 교차 확인)
4. `poe.ninja` (메타/시세)

데이터 파일 상단이나 커밋 메시지에 **기준 패치 버전을 명시**한다. (예: `// 3.29 기준`)

---

## 4. 작업 방식

- 번역 수정 시: 변경 전/후를 표로 정리해서 보고한다.
- 용어집을 수정했으면 무엇을 추가/수정했는지 명시한다.
- 수치·젬 레벨·링크 구성 등 빌드 데이터는 번역 작업 중 절대 변형하지 않는다.
